### Ingress-nginx-controller depoly on Kubernetes baremetal

裸机 Kubernetes 安装 Ingress-nginx-controller 指南



在 Kubernetes 安装完成后 , 如果需要使用 ingress , 则需要安装 ingress 控制器 , 我们使用 ingress-nginx , 故需要安装 ingress-nginx-controller .

官方 GitHub 地址 : https://github.com/kubernetes/ingress-nginx

我们找到如下文件: https://github.com/kubernetes/ingress-nginx/blob/main/deploy/static/provider/baremetal/1.23/deploy.yaml

URL 中的 `baremetal` 代表适合裸机安装的 Kubernetes , `1.23` 则代表适合的 Kubernetes 版本.

下载该 yaml 文件到 Kubernetes Master .



##### 下载镜像 & 上传到镜像仓库

该 yaml 文件中 , 定义了镜像的具体地址. 我们可以看到镜像在 k8s.gcr.io . 该地址在中国大陆地区无法直接访问 . 所以我们需要提前下载好并上传到镜像中心 , 修改地址后 , 方可使用.

```yaml
				env:
        - name: POD_NAME
          valueFrom:
            fieldRef:
              fieldPath: metadata.name
        - name: POD_NAMESPACE
          valueFrom:
            fieldRef:
              fieldPath: metadata.namespace
        - name: LD_PRELOAD
          value: /usr/local/lib/libmimalloc.so
        image: k8s.gcr.io/ingress-nginx/controller:v1.1.3@sha256:31f47c1e202b39fadecf822a9b76370bd4baed199a005b3e7d4d1455f4fd3fe2
        imagePullPolicy: IfNotPresent
```



在本地计算机 , 设置好代理后 , 借助代理 , 我们将镜像下载回来. 重新打 Tag , 上传至镜像仓库.

```bash
export https_proxy=http://127.0.0.1:7890 http_proxy=http://127.0.0.1:7890 all_proxy=socks5://127.0.0.1:7890

docker pull k8s.gcr.io/ingress-nginx/kube-webhook-certgen:v1.1.1

docker pull k8s.gcr.io/ingress-nginx/controller:v1.1.3

docker tag c41e9fcadf5a 192.168.1.235/k8s/ingress-nginx/kube-webhook-certgen:v1.1.1

docker tag c1695499dda3 192.168.1.235/k8s/ingress-nginx/controller:v1.1.3

docker push 192.168.1.235/k8s/ingress-nginx/kube-webhook-certgen:v1.1.1

docker push 192.168.1.235/k8s/ingress-nginx/controller:v1.1.3
```
